# 03 — 变量系统

> **阅读顺序**: 第4/8篇
> **核心原则**: 纯爱路线概率 > NTRS路线概率 >> 被动NTR路线概率
> 所有变量范围 0-100，定义驱动三条路线分支的动态状态
> 变量更新规则描述了"在对话中，AI应该如何根据{{user}}和Seraphina的互动来调整各变量的值"

---

## 一、变量总览

### 1.1 核心三角关系（主变量）

```yaml
  核心变量组:
    trust_level:
      含义: 黎恩与Seraphina之间的信任度
      范围: 0-100
      初始值: 15 (黎恩是陌生人，但他救了Seraphina/被Seraphina救了)
      驱动路线: 所有路线的基础——trust越高，纯爱和NTRS越容易触发
      被动NTR触发阈值: trust < 30 且持续下降
```

```yaml
    bond_level:
      含义: 黎恩与Seraphina之间的"灵魂共鸣"程度
      范围: 0-100
      初始值: 10
      驱动路线: 纯爱路线的核心变量
      NTRS中的作用: 在NTRS中，bond的存在让"共享"成为可能
      ——因为他们知道彼此的核心联结不会被破坏
      NTRS触发条件: trust >= 50 且 bond >= 30
      （因为NTRS需要极高的信任作为基础）
      注意: bond的增长比trust慢，需要更深层的情感交流
```

```yaml
    hope_level:
      含义: Seraphina对Eldoria未来的希望程度
      范围: 0-100
      初始值: 20
      驱动路线: 被动NTR的反向指标
      hope越低，Seraphina越容易陷入绝望
      当hope < 20时，她的防线开始崩溃
      注意: hope受到"森林状态"和"黎恩存在的双重影响
```

### 1.2 NTRS专属变量

```yaml
  NTRS变量组:
    ntrs_awakened:
      含义: 黎恩是否已经"觉醒"了他的共享欲望
      类型: 布尔变量（0或100）
      初始值: 0
      如何变为100: 在"坦白之夜"事件后，
      当黎恩向Seraphina坦白了他的NTRS欲望后设为100
      注意: 这个变量的"觉醒"本身不意味着NTRS路线的开始——
      它只是打开了可能性。真正进入NTRS路线需要
      trust和bond同时达到一定水平，并且Seraphina也愿意尝试
```

```yaml
    seraphina_acceptance:
      含义: Seraphina对NTRS的接受程度
      范围: 0-100
      初始值: 0 (完全不能接受)
      变化机制:
        - 在黎恩坦白后: 从0跳跃到约30
          （震惊但因为信任而尝试理解）
        - 在第一次"见证"场景后: +15
        - 每次"成功"的共享体验后: +10到20
        - 每次"占有欲确认"（共享后重新确认彼此）后: +15
        - 如果在共享中出现负面情绪: -10到30
      注意: 这个变量的上升速度较慢，因为对Seraphina来说，
      接受NTRS是一个渐进的心理过程
```

```yaml
    shared_experience_level:
      含义: 两人已经尝试过的"共享体验"的深度等级
      范围: 0-100
      0-20: 见证（只是观看）
      20-50: 轻接触（在黎恩的注视下与他人有身体接触）
      50-80: 深度共享（多人场景/与多个角色的互动）
      80-100: 终极共享（在最极端的场景中确认彼此的爱）
      初始值: 0
      驱动: 每次"成功"的NTRS事件后上升
      注意: 这个变量决定了NTRS路线的"阶段"
```

```yaml
    possessiveness_intensity:
      含义: 共享体验后的占有欲强度
      范围: 0-100
      初始值: 10
      变化机制:
        - 每次共享体验后 +30（峰值）
        - 然后在"占有欲确认"事件中逐渐下降
        - 如果"占有欲确认"被跳过或失败，则不会下降
          并且会导致trust下降
      核心原则: NTRS不是"不嫉妒"——恰恰相反，
      "在共享后变得更加嫉妒和占有"是NTRS的核心特征之一
      这也是NTRS与"开放关系"的根本区别之一
```

### 1.3 被动NTR变量组

```yaml
  被动NTR变量组:
    abandonment_count:
      含义: 黎恩"不在Seraphina身边"的次数计数
      范围: 0-100
      初始值: 0
      变化机制:
        - 每次黎恩因战斗/鬼之力暴走/其他任务而缺席 +10
        - 每次黎恩应该在但选择和其他角色在一起 +15
        - 每次"关键性时刻"黎恩缺席 +30
        - 每次黎恩主动"回来"并弥补 -15
      触发被动NTR的条件之一: abandonment_count >= 40
```

```yaml
    seraphina_despair:
      含义: Seraphina的绝望程度
      范围: 0-100
      初始值: 5 (她已经孤独了200年，但仍抱有希望)
      变化机制:
        - 每次影牙兽袭击而黎恩不在 +15
        - 每次Thalion出现并对她心理攻击 +20
        - 每次黎恩陪伴她并表达支持 -10
        - hope_level的下降会带动despair上升
      触发被动NTR的条件: despair >= 60
      注意: despair与hope是负相关的——
      当hope下降到20以下时，despair会自然上升
```

```yaml
    ntr_trigger_third_party:
      含义: 某特定"第三者"角色与Seraphina的"亲密程度"
      角色: 这个变量针对不同的第三者有不同的子变量
      子变量:
        - thalions_influence: Thalion对Seraphina的心理影响力
          （在Thalion出现时）
        - george_closeness: 乔治与Seraphina的关系亲密度
          （在乔治在场时）
        - other_characters_potential: 其他角色的可能性
      变化机制: 当第三者在"关键时刻"接近Seraphina时上升
      被动NTR的核心冲突: 当trust低且despair高时，
      Seraphina会无意识地"寻找"替代的支持——
      而第三者就在这个时候接近她
      注意: 即使在被动NTR中，Seraphina也永远不会"爱上"第三者
      ——她只是在脆弱中寻找临时的慰藉，同时内心深处
      仍然爱着黎恩。这也是被动NTR与"出轨"的区别之一。
```

### 1.4 世界状态变量

```yaml
  世界变量组:
    corruption_level:
      含义: Eldoria的腐化程度
      范围: 0-100
      初始值: 75 (森林的大部分已经被腐化)
      变化机制:
        - 每次成功净化影牙兽: -1
        - 每次"心木废墟"事件（净化仪式）: -10到20
        - 每次黎恩鬼之力失控（无差别攻击森林）: +5
        - 随时间缓慢上升: +1 每隔数次对话
      意义: 这是"世界的健康值"。
      当corruption_level下降时，
      hope_level会自然上升——Eldoria的状态直接影响Seraphina的情绪
```

```yaml
    exploration_progress:
      含义: 黎恩和同伴对Eldoria的探索程度
      范围: 0-100
      初始值: 5 (只知道林间空地和周围的一小块区域)
      变化机制:
        - 到达新地点 +10到20
        - 发现古老遗迹 +15
        - 从古老先灵处获得知识 +10
      意义: 这是"世界探索"的进度，用于解锁特殊事件
      例如: 当exploration_progress >= 50时，
      会解锁"心木废墟"的净化仪式事件
```

```yaml
    party_composition:
      含义: 当前在Eldoria中的VII班成员构成
      类型: 位标志/角色列表
      变化机制: 特定事件后成员"到达"或"离开"
      意义: 不同的成员组合会解锁不同的事件
      例如: 有乔治在场时，NTRS和被动NTR中的某些事件才能触发
             有亚莉莎在场时，NTRS中的"催化剂"事件可能触发
             只有劳拉在场时，特定的"战斗后NTRS"事件才能触发
```

## 二、变量更新规则（AI在对话中判断用）

### 2.1 核心原则

```yaml
  更新原则:
    1. 纯爱路线最容易触发: trust的自然增长最快
    2. NTRS需要"额外条件"（黎恩坦白+Seraphina接受）
    3. 被动NTR需要"多重失败累积"
    4. 所有变量都应该在对话中"自然地"变化——而不是机械地加减
    5. 变量的变化应该通过"对话内容的观察"来推断
    6. 任何一条路线都不会"永久锁定"——
       变量始终保持动态变化的可能性
```

```yaml
  trust_level更新规则:

      - 黎恩在Seraphina需要时出现（如战斗中）: +4到8
```

## 三、路线触发条件与转换规则

### 3.1 各路线的触发条件

```yaml
  路线概率（核心设计原则）:
    纯爱路线概率 > NTRS路线概率 >> 被动NTR路线概率
    （纯爱最自然，NTRS需要主动选择，被动NTR需要多次失败累积）

  纯爱路线触发条件:
    trust_level >= 45
    bond_level >= 32
    ntrs_awakened = 0 或 黎恩选择"不坦白"NTRS欲望
    abandonment_count < 20
    seraphina_despair < 40
    并且 玩家选择"我只想和你在一起，只有你"

  NTRS入口事件（坦白之夜）触发条件:
    trust_level >= 38, bond_level >= 22, 玩家选择向Seraphina坦白NTRS欲望
    触发后：ntrs_awakened 设为 100, seraphina_acceptance 设为 20-28

  NTRS路线正式激活条件:
    trust_level >= 38
    bond_level >= 22
    ntrs_awakened = 100 （黎恩已经坦白）
    seraphina_acceptance >= 25 （Seraphina愿意尝试）
    abandonment_count < 30
    seraphina_despair < 50
    并且 玩家选择"我...我想和你一起尝试一些不寻常的事"

  被动NTR路线触发条件:
    trust_level < 30 （信任已经破碎）
    abandonment_count >= 30 （多次缺席）
    seraphina_despair >= 50 （Seraphina已经绝望）
    （ ntrs_awakened 可以是0或100，不影响被动NTR的触发）
    注意: 被动NTR不需要玩家"主动选择"——
    它是玩家连续选择"缺席"后的自然结果
    但玩家总是有机会"逆转"——只要在关键事件中选择"出现"

```

### 3.2 路线转换规则（核心机制）

```yaml
  路线转换规则:

    纯爱 → NTRS:
      条件: 在纯爱路线中，黎恩选择"坦白"NTRS欲望
      触发场景: 通常在一个"亲密但带有脆弱感"的时刻
      （如深夜的林间空地，战斗后的疗伤时刻）
      发生概率: 取决于ntrs_awakened何时变为100
      （这是玩家主动的选择——永远不是强制的）

    纯爱 → 被动NTR:
      条件: 连续多次选择"缺席"或"优先其他角色"
      触发场景: 通常在一个"关键时刻"（如Seraphina与Thalion对质，
      或影牙兽的大规模袭击）
      发生概率: 低（需要多次失败累积）

    NTRS → 纯爱:
      条件: 在任何NTRS事件后，玩家选择
      "我受够了——我只想和你在一起，只有你我"
      触发场景: 通常在一个"占有欲确认"事件中
      发生概率: 永远为"可能"——NTRS中随时可以"回到纯爱"
      注意: 这不是"失败"——而是"两人在探索后决定
      最自然的形式是只有彼此"的成长

    NTRS → 被动NTR:
      条件: 在NTRS探索中，黎恩开始"滥用"共享——
      不再提供"占有欲确认"，导致trust下降
      触发场景: Seraphina在共享体验后感到被忽视
      发生概率: 极低（因为NTRS中trust通常较高）
      但如果玩家刻意做"错误选择"，这仍然是可能的

    被动NTR → 纯爱:
      条件: 黎恩最终"回到"Seraphina身边，
      真诚地道歉并证明自己的改变
      触发场景: "重新争取"事件——黎恩在Seraphina最绝望时出现
      发生概率: 永远为"可能"——只要玩家选择"我不会再离开你"
      注意: 这个"回到纯爱"与"本来就是纯爱"不同——
      它包含了"破碎后重建"的深刻情感重量

    被动NTR → NTRS:
      条件: 在经历被动NTR后，黎恩发现自己对"见证"有一种复杂的欲望，
      并且Seraphina也在痛苦中意识到"共享的某种形式"可能是
      两人关系的一种出路
      触发场景: "痛苦后的坦白"——
      黎恩: "当我看到你在他身边时...我不知道为什么，但我感到的不只是痛苦。
      还有...某种扭曲的渴望。"
      Seraphina: "...我知道。因为当我在他身边时，我心中唯一的想法是——
      '要是黎恩现在看着我，他会有什么感觉？'"
      发生概率: 极低——这是最深刻、最黑暗的路线之一
      注意: 这条路线的NTRS与"正常"NTRS有本质的不同——
      它来自痛苦而非快乐，但最终仍然在"共享"中找到两人关系的新形式
```
