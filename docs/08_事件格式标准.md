# 08 — 事件格式标准

> **版本**: v1.0.0 | **阅读顺序**: 第9/9篇
> 本文档定义所有事件条目的统一YAML格式规范。新增或修改事件时，必须遵循本标准。

---

## 一、统一模板

每个事件条目由**标题行**和**YAML块**组成：

```
### 事件{ID}：{事件全称}

```yaml
  事件: {ID} {事件全称}
    触发条件: {变量} >= {值}, {变量} >= {值}, ...
    情境:
      {场景叙述。2空格缩进（相对于"情境:"键）。
       多段落续行使用相同缩进。}
    路线分支:  # 仅多路线事件
      纯爱路线: {内容与变量}
      NTRS路线: {内容与变量}
      被动NTR路线: {内容与变量}
    玩家选择:  # 仅需玩家交互的事件
      A. {选项} → {效果}
      B. {选项} → {效果}
    性行为等级: {1-12}  # 必填：S1-S41, C1-C6, N3-N15, P13-P16, PN11-PN21
    情感阶段: {A/B/C/D} — {动机简述}  # 必填：所有NTRS路线事件
    黎恩知情: {是/否/视分支} — {简述}  # 必填：所有NTRS路线事件及被动NTR NSFW
    变量更新:  # 始终使用无序列表
      - {var_1} +{delta}
      - {var_2} +{delta}~{range}
    核心: {事件总结/关键洞察}  # 可选
```
```

## 二、字段规范

### 2.1 字段层级

| 层级 | 字段 | 必填 | 适用事件 |
|------|------|:--:|---------|
| **基础** | `事件:` | ✅ | **全部** |
| **基础** | `触发条件:` | ✅ | **全部** |
| **基础** | `情境:` | ✅ | **全部** |
| **基础** | `变量更新:` | ✅ | **全部**（有变量变化的事件） |
| **分支** | `路线分支:` | 按需 | 多路线事件（三线各有独立分支） |
| **分支** | `玩家选择:` | 按需 | 含交互选项的事件 |
| **NSFW** | `性行为等级:` | ✅ | S/C/N(含NSFW)/PN(含NSFW)/P(NSFW) |
| **NTRS** | `情感阶段:` | ✅ | N系列、S系列NTRS分支、C系列 |
| **NTRS** | `黎恩知情:` | ✅ | N系列、S系列NTRS分支、PN NSFW |
| **扩展** | `所属章节:` | 可选 | 任一事 |
| **扩展** | `前置事件:` | 可选 | 有依赖链的事件 |
| **扩展** | `后续事件:` | 可选 | 有后续钩子的事件 |
| **总结** | `核心:` | 可选 | 任一事 |

### 2.2 字段说明

**`事件:`**
- 格式：`{ID前缀}{编号} {事件全称}`
- 示例：`P1 深夜的火炉边对话`、`S16 银流河畔的初次手交`
- 编号与标题行一致

**`触发条件:`**
- 变量条件用逗号分隔
- 多路线时分别列出：`纯爱路线: trust>=60, bond>=50 / NTRS路线: ntrs_awakened=100, acceptance>=50`
- 不在此字段中写叙事文本

**`情境:`**
- 场景叙述。可包含子区块：
  - `  场景:` — 初始场景设置
  - `  发展:` — 动作/进展
  - `  事后:` — 事后余韵/对话
- 多路线分支在 `路线分支:` 中处理，不在 `情境:` 中内联

**`路线分支:`**
- 三种标准键：`纯爱路线:` / `NTRS路线:` / `被动NTR路线:`
- 每条路线后跟内容和变量变化的简洁描述
- 不要用A/B/C标记路线分支

**`玩家选择:`**
- 选项用 `A.` / `B.` / `C.` 编号
- 效果用 `→` 连接
- 多个效果用逗号分隔

**`性行为等级:`**
- 1=裸露, 2=摸乳, 3=指交, 4=手交, 5=足交, 6=口交, 7=乳交, 8=腿交, 9=插入, 10=隐奸, 11=群交, 12=当面
- SFW事件不填此字段

**`情感阶段:`**
- 格式：`{阶段} — {动机简述}`
- 阶段值：A（紧张试探）、B（发现吃醋有趣）、C（放开享受+大胆挑逗）、D（自信掌控+庆祝）
- 可带箭头表示过渡：`B→C`
- 动机简述一句话：她为什么在此刻做这件事

**`黎恩知情:`**
- 格式：`{是/否/视分支而定} — {简述}`
- 简述：黎恩是否在场、是否在观看、是否知道

**`变量更新:`**
- 始终使用无序列表格式（`  - var +delta`）
- 不使用逗号分隔的内联格式
- 可标注范围：`+5~10`

**`核心:`**
- 一句话总结事件的关键情感/叙事功能
- 示例：`守护者契约的灵肉合一完成`、`隐奸——她已习惯被注视，黎恩隐身观看`

## 三、字段名迁移表

执行格式标准化时，将以下旧名替换为新统一名：

| 旧名 | 新统一名 | 说明 |
|------|---------|------|
| `变量:` | `变量更新:` | 仅当该键用于变量变化列表时 |
| `变量更新（核心）:` | `变量更新:` | 移除括号限定词 |
| `场景描述:` | `情境:` | 场景叙述 |
| `情境:` | `情境:` | 保持不变 |
| `场景细节:` | 合并入`情境:` → `  发展:` | 作为子区块 |
| `情境发展:` | 合并入`情境:` → `  发展:` | 作为子区块 |
| `事后:` | 合并入`情境:` → `  事后:` | 作为子区块 |
| `分支:` | `路线分支:` | 指路线变体时 |
| `后续玩家选择:` | `玩家选择:` | 合并入已有玩家选择区块 |
| `叙事注意:` | `核心:` | 叙事总结 |
| `场景氛围:` | `核心:` | 氛围总结 |
| `情感核心:` | `核心:` | 情感总结 |
| `核心设计:` | `核心:` | 设计总结 |
| `核心反差:` | `核心:` | 反差总结 |
| `影响:` | 合并入`变量更新:` | 旧版字段 |
| `后续:` | 合并入`情境:` | 后续叙事 |
| `事件基础效果:` | 合并入`变量更新:` | 旧版字段 |
| `NSFW等级:` | `性行为等级:` | C系列旧名 |

## 四、缩进规范

```
```yaml
  事件: ...           # 2空格（相对围栏）
    触发条件: ...     # 4空格
    情境:             # 4空格
      场景叙述文本。   # 6空格（值内容）
      续行文本。       # 6空格
    路线分支:         # 4空格
      纯爱路线: ...   # 6空格
      NTRS路线: ...   # 6空格
    玩家选择:         # 4空格
      A. ... → ...   # 6空格
    变量更新:         # 4空格
      - var +10      # 6空格（列表项）
      - var +5~8     # 6空格
    核心: ...         # 4空格
```
```

规则：
- 每级缩进 = 2空格
- 键在列4（相对围栏缩进2+2）
- 简单值内容在列6
- 列表项在列6
- 无Tab字符
- 无尾部空格

## 四-A、情境bullet写作规范

`情境:`字段内容使用 **bullet大纲体**，禁止小说体段落。

### 4A.1 bullet规则

| # | 规则 | 说明 |
|---|------|------|
| 1 | 每条bullet 1-3句话 | 不超过80字/条 |
| 2 | 单句说完 | 不用破折号（——）补充解释，用逗号句号 |
| 3 | 不用括号补充 | 补充说明直接写进正文 |
| 4 | 对话叙事化 | 不逐字复述对话原话，叙事化总结 |
| 5 | 全事件4-8条bullet | 超过8条必须合并精简 |
| 6 | 不写元叙事总结 | 禁止"从第一次X到后来Y""这是递进的终点""信任扩容到了边界" |
| 7 | 不写"不是X而是Y" | 直接写Y，不需用否定X来衬托 |
| 8 | 相邻bullet不同义重复 | 两句话能合并为一句且不丢失信息，必须合并 |

### 4A.2 bullet结构时序

```
bullet 1:   起因+场景设定（为什么此刻此地此人）
bullet 2-4: 上升——渐进身体接触，每步有心理伴随
bullet 5-6: 高潮——最强体验+内心独白
bullet 7:   余韵——事后第一句话/第一个眼神/身体痕迹
占有欲确认: 黎恩反应+菲娜回应→回到彼此
```

### 4A.3 好/坏对比

```
❌ "她跪下去之前手指攥紧了黎恩的手——然后松开。走到艾德里安面前的每一步膝盖都在发软，但步子没有停。不是害怕——是紧张。"
✅ "她跪下去之前攥紧黎恩的手，然后松开。走到艾德里安面前的每一步膝盖都在发软，但步子没停。"

❌ "从第一次面对一只低语者时的恐惧到今天咽下第一口的接受，这不是堕落，是信任的扩容。"
✅ "从第一次面对一只低语者时的恐惧到今天咽下第一口。信任扩容到了她无法想象的边界。"
```

### 4A.4 小说体 vs bullet体对比

```
❌ 小说体（禁用）:
  深夜的木屋里只有炉火的余烬在跳动。黎恩坐在窗边的椅子上，窗帘没有拉上。
  外面是Eldoria的夜，虫鸣和远处银流河的水声交织在一起。他拉过菲娜让她跪在
  自己腿间。她没有问他为什么选在窗边，只是抬头看了他一眼...

✅ bullet体（必须）:
  - 深夜木屋。黎恩坐窗边椅子上，窗帘没拉。外面虫鸣和远处银流河水声。
  他拉过菲娜让她跪在腿间。她没有问为什么是窗边，只看他一眼——紧张里有信任
  - 她含住他的肉棒。不是第一次口交，但窗边是第一次。嘴唇裹紧龟头慢慢往下吞...

## 六、新增事件检查清单

新增事件时，逐项检查：

- [ ] 标题行 `### 事件{ID}：{全称}` 位于YAML围栏之前
- [ ] `事件:` 字段包含 `{ID} {全称}`
- [ ] `触发条件:` 使用逗号分隔的变量条件，多路线分别列出
- [ ] `情境:` 包含场景叙述（非空）
- [ ] 多路线事件：`路线分支:` 含 `纯爱路线:`/`NTRS路线:`/`被动NTR路线:`
- [ ] 交互事件：`玩家选择:` 含A/B/C选项
- [ ] NSFW事件：`性行为等级:` 使用1-12等级值
- [ ] NTRS事件：`情感阶段:` 含阶段字母+动机
- [ ] NTRS/PN NSFW事件：`黎恩知情:` 含知情状态+简述
- [ ] `变量更新:` 使用无序列表格式
- [ ] YAML缩进：2空格/级
- [ ] 无Tab，无尾部空格
- [ ] 无旧字段名（对照第三节迁移表）
- [ ] 无嗅觉描写（全项目铁律）
- [ ] `核心:` 可选，如填写则一句话总结

## 七、禁止项

1. ❌ 使用旧字段名（`场景描述:`、`叙事注意:`、`变量:`、`分支:`等）
2. ❌ 变量更新使用内联逗号格式（`变量更新: trust+5, bond+3`）
3. ❌ 标题行与YAML `事件:` 字段中的编号不一致
4. ❌ NSFW事件缺少 `性行为等级:`
5. ❌ NTRS事件缺少 `情感阶段:` 或 `黎恩知情:`
6. ❌ 嗅觉描写（全项目铁律）
7. ❌ 路线分支使用A/B/C而非纯爱/NTRS/被动NTR
8. ❌ YAML围栏缺失或不匹配
