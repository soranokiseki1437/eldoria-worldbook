# 事件TXT模板填写规则

> 参照模板：`_TEMPLATE.TXT`
> 权威格式标准：`../docs/08_事件格式标准.md`
> 写作铁律：`../../CLAUDE.md`（第三、四节）

---

## 一、文件名与ID规则

| 规则 | 说明 |
|------|------|
| 文件名 | `{PREFIX}{number}.TXT`，如 `N15.TXT`、`E1.TXT` |
| 前缀选择 | E(阶段零) / P(纯爱) / N(NTRS) / PN(被动NTR) / W(世界) / H(隐藏) / G(通用SFW) / R(黎恩专属) |
| 编号 | 新事件先给临时高编号（如N99），然后跑 `renumber_events.py` 重排 |
| ID行 | 第一行必为 `ID: {PREFIX}{number}`，与文件名一致 |

## 二、字段填写规则

### 必填字段（所有事件必须有）

| 字段 | 格式 | 示例 |
|------|------|------|
| `ID` | `{PREFIX}{number}` | `N15`、`E1`、`P3` |
| `名称` | 中文，10-20字，简洁有力 | `坦白之夜——路线分支点` |
| `触发` | 变量阈值 / 前置事件 / 场景条件，逗号分隔 | `trust>=50, bond>=40, P19已触发` |
| `NSFW` | `是` 或 `否` | `是` |
| `情境` | 每行 `- bullet`，至少3条，按时间顺序 | 见模板 |

### 情境 bullet 写作规范

```
1. 环境/状态/情绪铺垫（1-2条）→ 在哪里、什么状态、为什么此刻发生
2. 触发时刻/身体反应/对话（2-4条）→ 核心动作、感官细节、关键对话
3. 高潮/转折（1-2条）→ 最强体验或转折点
4. 余韵/后果（1-2条）→ 心理回应、身体痕迹、变量暗示
```

- 每条1-3句话，第三人称叙事
- 用角色名字或代词（菲娜/Seraphina、黎恩/Rean、第三者本名）
- 按时间顺序排列，不在一条bullet里跳时间
- 不写"成本""效率""优化"等商业词汇（去AI化铁律#1）
- 不括号堆砌补充解释（去AI化铁律#2）
- 不出现事件编号引用如N##（去AI化铁律）

### 条件字段（按路线/类型决定是否填写）

| 字段 | 何时需要 | 格式 |
|------|---------|------|
| `性行为等级` | NSFW事件 | 数字 0-10。0=无性行为，1-3=轻度，4-6=中度，7-10=重度 |
| `情感阶段` | N/PN事件 | `A（标签）` / `B（标签）` 等。NTRS: A-E，被动NTR: A-D+ |
| `黎恩知情` | N/PN事件，性行为等级>0 | `是——{情境说明}` / `否——黎恩不知情` / `N/A——黎恩不在场` |
| `第三者` | N/PN事件，有第三者参与 | `{姓名}（{身份}）`，如 `凯尔（学者）`。无则写 `无` 或不填 |
| `占有欲确认` | NTRS NSFW事件 | bullet叙事，事后黎恩对菲娜的占有欲爆发 |
| `玩家选择` | 有分支互动的事件 | `A/B/C` 选项 + `→` 变量影响。纯线性事件不填 |
| `变量` | 几乎所有事件 | `变量名+数值` 或 `变量名=数值`，逗号分隔。参考 `03_变量系统.md` |
| `核心` | 几乎所有事件 | 1-2句话：事件设计意图、叙事位置、在路线中的作用 |

### 字段顺序

```
ID → 名称 → 触发 → NSFW → 性行为等级 → 情感阶段 → 黎恩知情 → 第三者
→ 占有欲确认 → 情境 → 玩家选择 → 变量 → 核心
```

不用的条件字段直接整行删除，不留空行。

## 三、各前缀特殊规则

| 前缀 | 特殊字段 | 不需要的字段 | 备注 |
|------|---------|-------------|------|
| E | — | 性行为等级、情感阶段、第三者、黎恩知情、占有欲确认 | 阶段零共通，纯SFW |
| P | 性行为等级（NSFW子系列P17-P22）| 情感阶段、第三者、黎恩知情 | 纯爱独占 |
| N | 情感阶段、性行为等级、黎恩知情、第三者、占有欲确认 | — | 最完整的字段集 |
| PN | 情感阶段、性行为等级、黎恩知情、第三者 | 占有欲确认 | 被动NTR无占有欲确认 |
| W | — | 性行为等级、情感阶段、第三者 | 世界事件 |
| G/H/R | — | 性行为等级、情感阶段 | 通用/隐藏/黎恩专属 |

## 四、新增事件工作流

```
Step 0: 确认前缀和编号
  ├── 确定事件属于哪个前缀（E/P/N/PN/W/H/G/R）
  ├── 新事件给临时高编号（如 N99），避免和现有冲突
  └── 文件名：{PREFIX}99.TXT → 放入 docs/event/{PREFIX}/

Step 1: 填充内容
  ├── 复制 _TEMPLATE.TXT → 重命名为你的文件
  ├── 按本文规则填写所有必填字段
  ├── 删除不适用的条件字段（整行删除）
  └── 参照同前缀已有TXT的质量标准

Step 2: 注册到章节体系
  ├── 编辑 scripts/assign_chapters.py → DEFAULT_CHAPTERS
  └── 将新事件ID加入对应章节的 events 列表

Step 3: 重编号（填补空缺）
  └── python scripts/renumber_events.py {PREFIX}

Step 4: 全流程重建
  └── python scripts/rebuild_all.py

Step 5: 验证
  └── python scripts/event_tool.py validate
```

## 五、范例

### SFW事件（E1 林间空地的苏醒）

```

名称: 林间空地的苏醒
触发: 游戏开始 / 重置
NSFW: 否
情境:
  - 黎恩在温暖的毛皮上醒来，周围是简陋但整洁的木屋...
  - 一位粉发琥珀色眼睛的女性正在火炉边照料草药...
  - "你醒了。你在森林中昏迷不醒，鬼之力失控..."
玩家选择:
  A. 诚实回答 → trust +5, bond +3
  B. 保持警惕 → trust +3
变量: exploration_progress +3
核心: 游戏开场。Seraphina介绍自己为"这片森林的守护者"。
```
